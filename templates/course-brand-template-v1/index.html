Outcomes

學完後，頁面應該讓人看見具體能力

01

定位課程價值

把課程名稱、受眾、痛點與可交付成果排在前段，讓訪客先理解這門課為何存在。

02

建立信任路徑

用學習節奏、講師證據、學員回饋與 FAQ 逐步降低報名疑慮。

03

支援重複產出

品牌 token 與版面規則分離，後續可替換課程資料而不重新發明視覺風格。

Learning Path

用四個階段說清楚學習推進方式

  1. Week 1-2

    釐清目標與受眾

    定義學員的起點、目標職能與課程可承諾的結果。

  2. Week 3-6

    完成核心專案

    用任務制拆解實作，讓學員每週都有能被檢查的產出。

  3. Week 7-10

    整合作品集

    把過程、決策、結果整理成可展示的案例敘事。

  4. Week 11-12

    Demo 與回饋

    透過發表、講師回饋與修訂清單完成最後打磨。

Curriculum

課綱用模組呈現，方便比較也方便替換

每個模組都應包含任務、交付物與學員能拿走的成果，而不是只列課程標題。

Module A

品牌與招生訊息

整理課程定位、受眾語言、痛點與首頁 hero 的主要承諾。

Module B

學習任務與成果證據

設計課程亮點、階段任務、成果展示與學員回饋的資訊架構。

Module C

招生決策與 FAQ

補上費用、時間、程度、支援方式等報名前最常見的阻力。

FAQ

把報名前的阻力整理成可展開的答案

這個模板有包含報名表單嗎？

沒有。這個 chat 的範圍只包含前端模板與互動，不定義 form schema，也不接任何後端。

可以替換成不同課程品牌嗎？

可以。請優先調整 `styles/course-brand-template-v1.json` 和 token，再微調頁面內容。

動畫是否支援 reduced motion？

支援。使用者偏好減少動態時，頁面會停用 reveal 位移與 hero 浮動效果。
